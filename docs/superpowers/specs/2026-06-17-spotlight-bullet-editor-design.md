# Spotlight Bullet 编辑器设计规格

**日期**: 2026-06-17
**状态**: 已批准（UX 层面）
**范围**: 为现有 Spotlight 新增多级 bullet list 编辑器模式，用于快速拆分/重组/切换任务

---

## 1. 目标与动机

当前 Spotlight 是单行命令输入（`!` `@` `+` `%` `>`），适合精确的快速操作，但不擅长：

- **随时拆分任务**：发现一个任务需要细化成多个子任务时，必须多次输入 `!子任务:父任务搜索词`
- **建立前置条件**：想到"做这件事之前先要做那件事"时，无法直观地把前置任务插入到正确层级
- **鸟瞰全局**：单行输入只能看到搜索建议，看不到任务间的父子关系

**解决思路**：新增一个"bullet 编辑器模式"，像 Workflowy/Logseq 那样以多级 bullet list 展示所有活跃任务，支持键盘快速建子任务/同级任务、单击切换 focus、原地编辑标题。

---

## 2. UX 决策汇总

| 维度 | 决策 |
|------|------|
| 集成方式 | 新增编辑器模式，保留原命令输入 |
| 任务范围 | 所有活跃任务（queued + waiting + active）的完整层级树 |
| 模式入口 | 输入框为空时按 `Tab` 进入；`Esc` 退出回到命令输入 |
| Focus 按钮 | 每行最左侧 `▶`，单击即切换 focus（自动停止当前 focus） |
| 标题编辑 | 单击标题原地编辑，失焦自动保存 |
| 完成 | `Ctrl+Enter` 或悬停 `⋯` 菜单 |
| 删除 | `Ctrl+Backspace` 或悬停 `⋯` 菜单 |
| 排序 | 不支持手动上下移动（YAGNI，按 `sort_order`/`created_at` 自动排序） |

## 3. 键盘交互模型（Option A — 用户原始描述）

在 bullet 编辑器模式内：

| 按键 | 行为 |
|------|------|
| `Enter` | 新建**子任务**（缩进 +1，光标移到新空行） |
| `Shift+Enter` | 新建**同级任务**（同缩进，光标移到新空行） |
| `Backspace`（行首且无内容） | 先取消缩进一级；已在顶层则删除该空行 |
| `Tab` | 手动缩进 +1（将当前行变成上方任务的子任务） |
| `Shift+Tab` | 手动取消缩进 -1 |
| `Ctrl+Enter` | 完成当前行任务（归档） |
| `Ctrl+Backspace` | 删除当前行任务（从数据库删除） |
| `Esc` | 退出编辑器模式，回到命令输入 |
| `↑` `↓` | 在任务行之间移动光标 |
| 单击标题 | 进入原地编辑 |

### 空行行为

- 空行 `Enter` = 在当前位置新建子任务（缩进+1）
- 空行 `Shift+Enter` = 新建同级
- 空行 `Backspace` = 取消缩进；已在顶层再按则删除空行并把光标移回上一行

### 边界情况

- **新建子任务时的项目继承**：自动继承父任务 `project_id`
- **删除有子任务的任务**：子任务一并删除（递归级联）。⚠️ 现有 `deleteTask` (`electron/db/service.ts:137`) 只删单条，需扩展为递归删除所有后代，或删除前先把子任务的 `parent_id` 重新指向被删任务的父级（re-parent）。**推荐递归删除**，并在悬停菜单/右键菜单操作前提示"将同时删除 N 个子任务"
- **完成有子任务的任务**：仅完成自身，子任务保持不变（用户可选择后续手动完成）
- **focus 一个有子任务的任务**：只 focus 该任务自身，子任务不变

## 4. 视觉布局

```
┌─────────────────────────────────────────────────────────────┐
│  🔍  [输入框]                                        [BULLET] │
├─────────────────────────────────────────────────────────────┤
│  ● 当前 Focus  （仅当有 active task 时显示）                │
│  ┃ ▶  ● 写论文                              2h30m  ●●●      │
│  ┃                                                       │
│  ● 任务树                                                   │
│     ▶  🟦 跑步锻炼                                    ⋯    │
│     ▶  🟪 读论文                                   ⋯    │
│     ┃  ▶  🟪 精读 BERT                            ⋯    │
│     ┃  ▶  🟪 精读 GPT4  ← (正在编辑)              ⋯    │
│     ┃  ▶  🟪 写实验代码                             ⋯    │
│     ▶  🟩 买菜                                    ⋯    │
│                                                           │
├─────────────────────────────────────────────────────────────┤
│  Enter=子任务  Shift+Enter=同级  Esc=退出                   │
└─────────────────────────────────────────────────────────────┘
```

### 视觉元素

- **Focus 按钮**（`▶`）：最左侧，hover 高亮，颜色跟随项目色；当前 focus 中的任务显示为 `▶` 实心 + 高亮背景
- **项目色块**（🟦🟪🟩）：每行标题前，3-4px 色条或圆点，对应 `project.color`，无项目的显示灰色
- **当前编辑行**：淡色高亮背景（如 `bg-white/5`），标题变为可编辑 input
- **悬停 `⋯` 按钮**：行右侧浮现，点击展开菜单（完成 / 删除 / 添加备注 / 修改项目 / 设定计时器）
- **层级连接线**（`┃`）：用左边框模拟，比纯缩进更清晰
- **计时显示**：focus 中任务显示 `+ HH:MM:SS`，waiting 任务显示 `- 倒计时`

### 滚动与展开

- 默认展开到当前 focus 任务的路径，其他分支折叠（仅显示根任务）
- 点击项目符号（`▶`/`▼`）切换展开/折叠
- 树过大时使用滚动，不需要虚拟化（活跃任务通常 < 100 条）

## 5. 模式切换

### 进入编辑器模式

- 条件：Spotlight 输入框内容为空（`input === ''`）
- 触发：按 `Tab` 键
- 反馈：输入栏右侧显示 `[BULLET]` 标签，输入栏 placeholder 变为 "Bullet 编辑器中… Tab 进入，Esc 退出"
- 渲染：Suggestions/Backlog 区域被 `SpotlightBulletEditor` 替换

### 退出编辑器模式

- 触发：按 `Esc`
- 反馈：回到命令输入模式，光标聚焦到输入框
- 任何正在编辑的标题立即保存（blur + save）

### 命令模式与编辑器模式的共存

- Active Zone（当前 focus + waiting tasks）始终显示在顶部
- Training tasks 行始终显示
- 仅 Suggestions 区域被编辑器替换
- Footer 快捷键提示根据模式切换

## 6. 数据与状态

### 复用现有数据结构

无需改动数据库 schema。现有 `tasks` 表已支持：

- `parent_id` — 父子关系
- `sort_order` — 排序
- `is_next_action` — 是否为下一步行动
- `project_id` — 项目归属
- `status` — 'active' | 'waiting' | 'queued' | 'archived'

### 复用现有 IPC

所有操作通过现有 `window.api` 完成：

| 操作 | IPC |
|------|-----|
| 新建任务（含父子） | `createTask(title, tag?, type?, projectId?, parentId?)` |
| 修改标题/项目/父级 | `updateTask(id, { title / project_id / parent_id })` |
| 删除任务 | `deleteTask(id)` |
| 完成 | `completeTask(id)` |
| 切换 focus | `startFocus(id)` |

### 树构建

- 输入：`tasks.filter(t => t.status !== 'archived')`
- 输出：按 `parent_id` 分组，根节点为 `parent_id === null`
- 同级排序：`sort_order` 升序，`created_at` 作为 tiebreaker
- 循环引用防护：构建时检测，断开异常边

## 7. 实现要点（概要）

### 新增文件

- `src/renderer/components/SpotlightBulletEditor.tsx` — 编辑器主组件
- `src/renderer/utils/buildTaskTree.ts` — 扁平 → 树（纯函数）
- `src/renderer/hooks/useOutlinerKeyboard.ts` — 键盘交互 hook

### 修改文件

- `src/renderer/components/Spotlight.tsx` — 添加 `editorMode` 状态、Tab 检测、条件渲染

### 关键实现细节

- **标题原地编辑**：使用 `<input>` 或 `contentEditable`，blur 时调用 `updateTask`，Enter 不触发保存（被键盘 hook 拦截用于新建子任务）
- **焦点管理**：维护 `activeNodeId` 状态（当前光标所在行），方向键和新建操作更新它
- **乐观更新**：本地先改 tree，再调 IPC；失败回滚并 toast 提示
- **IPC 监听**：继续监听 `onTimerUpdate`/`onFetchTasks` 同步外部变更
- **递归删除**：`deleteTask` 需扩展为递归（先查所有 `parent_id = id` 的子任务，深度优先删除），或新增 `deleteTaskRecursive` IPC。编辑器调用前先在本地 tree 上计算后代数量用于确认提示

## 8. 不做的事（YAGNI）

- ❌ 手动上下移动任务（Alt+方向键）— 用户明确未选
- ❌ 拖拽排序 — 复杂度高，键盘已足够
- ❌ 多选批量操作 — 当前需求未提及
- ❌ 虚拟化长列表 — 活跃任务量级不够
- ❌ 标签（tag）的复杂管理 — 现有命令模式已覆盖
- ❌ 撤销/重做 — 依赖数据库一致性，后续可加

---

## 9. 成功标准

1. 空输入按 `Tab` 能进入编辑器，看到所有活跃任务的层级树
2. 在任意任务上按 `Enter` 能立即创建子任务并开始输入
3. 点左侧 `▶` 能在 < 200ms 内切换 focus（含 IPC 往返）
4. 单击标题能原地编辑，失焦后刷新主窗口能看到新标题
5. `Esc` 能退出回到命令输入，不丢失任何状态
6. 现有命令模式功能（`!` `@` `+` `%` `>` `#`）完全不受影响
